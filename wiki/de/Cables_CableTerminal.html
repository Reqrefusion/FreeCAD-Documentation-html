Beschreibung

Der Anschluss, das CableTerminal-Objekt, ist ein abhängiges (integriertes) Objekt und kann nicht direkt erstellt werden.

Er stellt einen elektrischen Anschlusspunkt (für eine Ader) einer Leitung. Er kann als integriertes Objekt während der Erstellung einer Leitungsverbindung oder eines Betriebsmittels erstellt werden. Seine Haupraufgabe ist, die Position für das Ende einer Ader innerhalb einer Leitungsverbindung oder enes Betriebsmittels vorzugeben. Müssen mehrere Adern an einem einzelnen Anschluss verbunden werden, kann in mehrere Anschlusselemente aufgeteilt werden, mit denen weitere Adern verbunden werden können. Das CableTerminal-Objekt ist vollständig parametrisch, d. h. dass seine Länge, Anzahl der möglichen Verbindungen und Positionierung relativ zu seinem übergeordneten Objekt eingestellt werden können.

Der Anschluss ist dafür vorbereitet, ein Bauteil in einer elektrischen Schaltung zu sein und besitzt die Eigenschaften Pin Name (Anschlussname), Node Name (Knotenname) und Connected Wires (Angeschlossene Adern), die in zukünftigen Ausgaben des Arbeitsbereichs Cables für die Analyse der Schaltung verwendet werden können. Jeder Anschluss stellt einen einzelnen Kontakt oder Anschlusspunkt innerhalb einer Leitungsverbindung oder eines Betriebsmittels dar.

Wichtig: Die tatsächliche Beziehung zwischen einem Anschluss und seinem übergeordneten Objekt ist umgekehrt, um zyklische Abhängigkeiten zu vermeiden und die Neuberechnung zu beschleunigen. Dies hat einige Konsequenzen (siehe Hinweise).

Beispiel von zwei Anschlüssen. Links: Ein einzelner Anschluss mit einem Anschlusselement. Nur ein Leiter kann hier angeschlossen werden. Rechts: Ein einzelner Anschluss mit drei Anschlusselementen. Bis zu drei Leiter können hier angeschlossen werden.

Anwendung

Der Anschluss, das CableTerminal-Objekt, kann nicht direkt erzeugt werden. Es wird während der Erstellung einer Leitungsverbindung oder eines Betriebsmittels erzeugt, entsprechend der Eigenschaft Number Of Terminals dieser Objekte.

Hinweise

  • Das CableTerminal-Objekt hat eine umgekehrte Beziehung zu seinem übergeordneten ElectricalDevice-Objekt, um zyklische Abhängigkeiten zu vermeiden. Es wird in der Baumansicht (nur deshalb) als untergeordnete Elemente angezeigt, um den Zusammenhang der Objekte darzustellen. Undesirable consequence: Unerwünschte Konsequenz: Wenn ein übergeordnetes Objekt kopiert wird, müssen alle seine untergeordneten Anschlüsse manuell mitkopiert werden.
  • Zum Anschluss gehören zwei Befehle: LeiterMitAnschlussVerbinden und LeiterVomAnschlussLösen. Sie ermöglichen ein WireFlex-Objekt mit einem Anschluss zu verbinden und wieder zu lösen. Es wird ausdrücklich empfohlen, dafür diese Befehle zu verwenden, da sie im Hintergrund erforderliche Anpassungen an den Eigenschaften der Anschlüsse vornehmen.
  • Die Eigenschaften eines Anschlusses werden während der Erstellung seines übergeordneten Objekts festgelegt. Einige von ihnen können später manuell angepasst werden.
  • Wenn eine Leitung auf einem verbundenen WireFlex-Objekt aufgebaut wird, muss der Wert der Daten-EigenschaftStripped Wire Length einer Leitung größer sein als die Länge eines Anschlusselements (Daten-EigenschaftLength). Einige andere Einstellungen der Leitungen und WireFlex-Objekte müssen eventuell angepasst werden (wie z. B. Boundary Segment Start oder Boundary Segment End), dammit die Verbindung korrekt dargestellt wird.

Eigenschaften

Daten

Attachment

  • DatenAttachment Offset (Placement): Enthält eine Positionierung relativ zum übergeordneten Objekt des Anschlusses. Sie wird während der Erstellung eines Anschlusses angelegt, kann aber den Bedürfnissen des Anwenders entsprechend angepasst werden.

Net

  • Daten-EigenschaftConnected Wires (StringList): (schreibgeschützt) Enthält eine Liste der verbundenen Adern mit der Nummer des Anschlusselements. Diese Eigenschaft wird automatisch aktualisiert, wenn ein Anschluss neu berechnet wird. Die verbundenen Adern sind nicht wirklich untergeordnete Objekte des Anschlusses; sie werden nur in der Baumansicht so dargestellt.
  • Daten-EigenschaftNode Name (String): Enthält den Namen des Anschlussknotens. Der Knotenname sollte für alle Anschlüsse, die mit elektrischen Leitern verbunden sind, identisch sein. Die Eigenschaft Node Name (Knotenname) bezieht sich auf (Leitungs-) Knoten (engl. https://en.wikipedia.org/wiki/Node_(circuits) node) in einer elektrischen Schaltung. In der aktuellen Version werden die Knotennamen, die Eigenschaft Node Name, an zwei verbundenen Anschlüssen nicht automatisch überprüft.
  • Daten-EigenschaftPin Name (String): Enthält den funktionellen Namen des Kontaktstiftes oder des Anschlusspunktes in einer Leitungsverbindung oder einem Betriebsmittel.

Terminal

  • Daten-EigenschaftLength (Length): Enthält die Länge der Anschlusselemente
  • Daten-EigenschaftNumber Of Connections (Integer): Enthält die Anzahl der Anschlusselemente des Anschlusses.
  • Daten-EigenschaftSpacing (Length): Enthält den Abstand zwischen Anschlusselementen des Anschlusses.

LuxRender wird nicht mehr weiterentwickelt, da das Projekt als LuxCoreRender neu gestartet wurde, mit einem wesentlich umgeschriebenen Code und vielen inkompatiblen Änderungen. Diese Information wird hier vorgehalten, da FreeCAD noch immer (Version 0.19-24276) mit dem Arbeitsbereich Raytracing ausgeliefert wird, der offiziell nur LuxRender unterstützt. Er scheint aber auch mit LuxCoreRender zu funktionieren, daher sollte man ihn ausprobieren, bevor man LuxRender verwendet

Beschreibung

LuxRender ist neben POV-Ray eine der beiden vom Arbeitsbereich Raytracing unterstützten Render-Engines. Im Jahr 2013 wurde das Projekt neu gestartet und in LuxCoreRender umbenannt, wobei der Code weitgehend neu geschrieben und kompatibilitätsbrechende Änderungen vorgenommen wurden, sodass das LuxRender-Projekt aufgegeben wurde. Offiziell wird LuxCoreRender vom Arbeitsbereich Raytracing nicht unterstützt, obwohl es einen Versuch wert sein könnte.

Installation

Arbeitsbereich Raytracing

Der Arbeitsbereich Raytracing wird abgelöst durch den neuen Arbeitsbereich Arbeitsbereich Render, der als Ersatz geplant ist. Der Arbeitsbereich Render kann mit dem Addon-Manager installiert werden. Diese Information wird bereitgestellt, weil FreeCAD als Standardeinstellung (mit Stand 0.19-24276) weiterhin mit dem Arbeitsbereich Raytracing ausgeliefert wird.

Stabile Version

Die neueste stabile Version ist LuxRender 1.6 (28.12.2017).

Linux

Kompilierte Binärdateien

As LuxRender is no longer developed, almost no distribution still has it in its repositories, so you will have to manually install the official binaries. First determine if your graphic cards supports OpenCL and if you have all the necessary dependencies installed. If so, download LuxRender 1.6 with OpenCL support. Otherwise download LuxRender 1.6 without OpenCL, but consider getting a newer graphic card or installing the necessary software to enable it on your computer.

The faster solution (although not the best practice) is to extract the content of the archive in a suitable location, like ~\LuxRender. If needed change the file permissions so that your user can execute the files you just extracted.

Aus dem Quellcode kompilieren

If your distributions does not have LuxRender in the repositories and the official binaries do not work on your distribution, or if you wish to, it is possible to compile LuxRender from source. Download the source code of LuxRender 1.6 from GitHub

FreeCAD konfigurieren

After installing LuxRender, launch FreeCAD, open the Preferences Editor, load the Raytracing Workbench, and go to the Raytracing Preferences.

Set the Luxrender executable path to point to your installation of LuxRender, in particular to the luxrender executable, and apply. In the above example, the path would be ~\LuxRender\luxrender.

macOS

Use the official LuxRender 1.6 installer for OSX.

Windows

First determine if your graphic cards supports OpenCL and if you have the necessary graphics drivers installed. If so, download LuxRender 1.6 with OpenCL support setup. Otherwise download LuxRender 1.6 without OpenCL setup, but consider getting a newer graphic card or installing the necessary software to enable it on your computer. Then execute the downloaded installer and follow the proposed steps.

Standardmäßig ist der Zielordner C:\Program Files\LuxRender, mit Beispielszenen in C:\Users\Public\Documents\LuxRender\Example Scene.

Nach der Installation von LuxRender startet man FreeCAD, öffnet den Voreinstellungseditor, lädt den Arbeitsbereich Raytracing und geht zu den Raytracing-Einstellungen.

Den Pfad zur ausführbaren Datei von Luxrender so festlegen, dass er auf die Installation von LuxRender verweist, in der Regel ist dies C:/Program Files/LuxRender/luxrender.exe, und man übernimmt die Einstellung.

Entwicklungsversion

Es gibt keine Entwicklungsversion von LuxRender, da die Entwicklung zugunsten des Neustarts von LuxCoreRender eingestellt wurde.

Arbeitsbereich Render

Der Arbeitsbereich Render unterstützt LuxRender nicht mehr, da es veraltet ist. Stattdessen unterstützt sie den modernen Neustart LuxCoreRender.

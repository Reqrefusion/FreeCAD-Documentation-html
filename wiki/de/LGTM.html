Übersicht

LGTM ist ein Code-Analyse-Tool, das in mehrere Online-Software-Versionskontrollsysteme integriert werden kann und verschiedene Sprachen unterstützt. Es ist ein hervorragendes Werkzeug zur Überprüfung der Codequalität, das Code-Probleme identifiziert, die von anderen Code-Checkern und Lintern oft übersehen werden.

LGTM eignet sich gut als Code-Analyse-Werkzeug für die Entwicklung von FreeCAD Python-Arbeitsbereichen und anderen kleinen bis mittelgroßen Projekten. Diese Seite bietet einen Überblick darüber, wie man LGTM mit einem FreeCAD Python-Arbeitsbereich verwenden kann.

Erste Schritte

Die ersten Schritte mit LGTM hängen von der verwendeten Online-Versionskontrollplattform ab. Die LGTM-Dokumentation für automatisierte Codeüberprüfung bietet einen guten Überblick darüber, wie man LGTM für verschiedene Plattformen in das Projekt integrieren kann.

Darüber hinaus ist es möglich, eine Vielzahl von detaillierten Code-Analysen auf LGTM durchzuführen, was jedoch den Rahmen dieses Tutorials sprengen würde. Weitere Informationen hierzu findet man in der LGTM-Dokumentation unter Konfigurieren der Codeanalyse.

Ergebnisse erhalten

Sobald man LGTM eingerichtet und Zugriff auf das Code-Repositorys gewährt hat, werden in der Regel täglich Analysen des Repositorys durchgeführt. Push-Änderungen führen also nicht sofort zu Ergebnissen. Es ist möglich, Pull-Anfragen bei ihrer Einreichung von LGTM analysieren zu lassen, wie in der LGTM-Dokumentation beschrieben.

Um die Ergebnisse zu überprüfen, muss man sich lediglich beim LGTM-Dashboard anmelden und das gewünschte Projekt auswählen. Von dort aus erhält man anhand der Code-Analysen eine Liste mit Problemen (wie Fehler, schlechte Codierungspraktiken, nutzloser/irrelevanter/unbenutzter Code usw.), die man sich ansehen kann. Darüber hinaus bietet LGTM eine Gesamtbewertung des Codes (A, B, C, D), die sich nach der Anzahl der Probleme im Verhältnis zur Gesamtgröße des Projekts richtet.

Probably, the most useful, immediate way to manage the results of your code analysis is simply filtering out files in your project that you don't want analyzed. That is, suppose you're developing new code that is incomplete, keeping around legacy code that is otherwise unused, or have a good deal of testing code that doesn't need analysis. LGTM provides file classification, an easy way to filter those files so they don't pollute your analysis results.

Creating a .lgtm.yml File

To enable file classification, first create a file named ".lgtm.yml" in your projects top-most directory. Then, in that file, add some classifications.

Nachfolgend findet man ein Beispiel aus dem Arbeitsbereich Python von FreeCAD-Trails: 

path_classifiers:
  template:
    - freecad/trails/resources
    - freecad/trails/project/commands/command_template.py
    - freecad/trails/init_gui.py
 
  test:
    - freecad/trails/project/commands/spiral_test.py
    - freecad/trails/TestFPO.py
 
  legacy:
    - freecad/trails/corridor/loft
    - freecad/trails/alignment/VerticalAlignment.py
    - freecad/trails/alignment/VerticalCurve.py
    - freecad/trails/alignment/GenerateVerticalAlignment.py
    - freecad/trails/alignment/ImportVerticalCurve.py
    - freecad/trails/corridor/template/TemplateLibrary.py

Man beachte, dass Einrückungsebenen in LGTM wichtig sind. Eine falsche Einrückung führt zu einer fehlerhaften Dateiklassifizierung.

Also, some classifications (like 'template' and 'test') are used by LGTM for queries and other analysis components. You may also define your own custom tags, which will filter code and provide additional queryable results.

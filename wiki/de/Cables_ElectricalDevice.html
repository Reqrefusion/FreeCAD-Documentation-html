Beschreibung

Das Elektrische Gerät stellt ein Modell eines beliebigen elektrischen Geräts dar, das über die Möglichkeit einer direkten Kabelverbindung verfügt. Seine Form kann aus der DatenVoreinstellungen-Liste bei der Erstellung ausgewählt werden. Es ist nur ein nicht parametrischer Formtyp verfügbar. Er wird als 'Fixiert' bezeichnet.

The 'Fixed' shape type creates non parametric device object with a shape taken from external STEP file. There are a few predefined STEP device models shipped together with Cables Workbench. Moreover, the user can create his own device library, this is described in Electrical Device Presets. In addition to the shape the certain number of CableTerminals are created with predefined size and position corresponding to connection points. Additionally one or more SuppLines objects can be created forming the basis for the cable entering the device.

Das Elektrische Gerät gehört zur Arch Komponenten-Klasse und erbt all ihre Eigenschaften.

Elektrische Geräte-Objekte können durch Zuordnen eines Materials zur Material-Eigenschaft eingefärbt werden. Mehrfach-Materialien werden ebenfalls unterstützt. Damit können die ursprünglichen Farben des Geräts in die in der Mehrfach-Materialzusammensetzung definierten Farben geändert werden. Wichtig: Die Anzahl der Farben im Mehrfach-Material muss mit der Anzahl der ursprünglichen Farben des Geräts übereinstimmen.

Anwendung

  1. Select the vertex, point on surface or object in the 3D View or select nothing.
  2. Create Electrical Device by one of these methods:
    • Press the Electrical Device button.
    • Select the Cables → Electrical Device option from the menu.
    • Right-click in the Tree View or the 3D View and select the Cables → Electrical Device option from the context menu.
    • Use the keyboard shortcut: C then E.
  3. The Electrical Device task panel opens. See Options for more information.

Einmal erzeugt, wird das Elektrische Gerät an der Position des ausgewählten Knotens, Punkts oder Objekts im 3D-Raum (oder am Punkt (0,0,0), wenn nichts ausgewählt wurde), platziert.

Optionen

Die im Aufgabenfenster verfügbare Voreinstellung Elektrisches Gerät kann geändert werden. Die aktuell ausgewählte Voreinstellung wird sofort in der 3D-Ansicht sichtbar.

Derzeit zeigt das Aufgabenfenster keine zusätzlichen Optionen an, wenn die spezielle Voreinstellung 'Angepasst' ausgewählt ist.

Voreinstellungen für elektrische Geräte

Der Arbeitsbereich Cables enthält mehrere vordefinierte Formen. Der Benutzer ist jedoch nicht auf die Verwendung des Standardsatzes beschränkt.

Eine zusätzliche CSV-Datei kann durch den Benutzer erstellt werden, die Vorlagen für selbsterstellte Arten von Elektrischen Geräten enthält. Sie muss devicepresets.csv heißen und hier gespeichert sein: 

$FREECAD_USER_DIR/Cables/

Das Verzeichnis $FREECAD_USER_DIR kann über die Python-Konsole abgefragt werden: 

FreeCAD.getUserAppDataDir()

Der Inhalt der benutzerdefinierten devicepresets.csv Datei muss nach den gleichen Regeln aufgebaut sein wie die devicepresets.csv im Arbeitsbereich Cables Quellcode. Die Datei ist selbsterklärend und kann als Vorlage für die Erstellung eigener Voreinstellungen verwendet werden.

Die benutzerdefinierte Datei devicepresets.csv kann geändert werden, ohne FreeCAD neu zu starten. Alle dort vorgenommenen Änderungen sind sofort in der Liste der Voreinstellungen sichtbar, wenn man über die Benutzeroberfläche ein neues elektrisches Gerät erstellt.

Aktuell kann die devicepresets.csv nur einen voreingestellten Typ enthalten: Fester voreingestellter Typ.

Fester voreingestellter Typ

Die Definition dieses Typs benötigt zwei weitere Dateien. Die Linie in der Datei devicepresets.csv definiert unter anderem die Namen dieser beiden zusätzlichen Voreinstellungsdateien:

  • STEP-Datei mit der Form und den Farben des Gerätes. Der Name dieser Datei muss direkt angegeben werden (Dateibeispiel: Dummy_Dummy.step)[1].
  • CSV-Datei mit Befestigungen des Gerätekörpers, Parameter für alle befestigten Kabelanschlüsse und Unterstützungslinien. Der Name dieser Datei wird aus der Verbindung der Felder DeviceClass und Name erstellt (Dateibeispiel: Device_Dummy.csv).

Diese beiden zusätzlichen Dateien (STEP und CSV) müssen im folgenden Verzeichnis abgelegt werden: 

$FREECAD_USER_DIR/Cables/lib

Tatsächlich sucht die Software an einem weiteren Ort nach diesen Dateien (der als schreibgeschützt behandelt werden sollte): 

$FREECAD_USER_DIR/Mod/Cables/freecad/cables/resources/presets/lib/

So ist es möglich, ähnliche Geräte mit derselben STEP-Definition, aber bei Bedarf mit einer unterschiedlichen Anzahl von Kabelanschlüssen oder deren Anordnung zu definieren.

Nach der Überprüfung findet man die Dummy_Dummy.step und Device_Dummy.csv Dateien unter $FREECAD_USER_DIR/Mod/Cables/freecad/cables/resources/presets/lib/.

Als Ergebnis der Definitionen aus diesen drei Dateien kann das neue Elektrische Gerät mit zwei Kabelanschlüssen und zwei Unterstützungslinien erzeugt werden.

STEP-Quelle

Jede STEP-Datei aus einer vordefinierten Voreinstellung hat auch ihre Quelldatei angehängt (Beispiel: Dummy_Dummy.FCStd). In jeder Quelldatei befinden sich Informationen darüber, wie eine STEP-Datei mit Farben aus der Quellform korrekt generiert werden kann. Diese Quelldateien können als Grundlage für die Erstellung von Formen für neue Geräte verwendet werden.

Hinweise

  • The CableTerminal and SuppLines objects have reversed relation with their Electrical Device parent to avoid cyclic dependencies. They are shown as a children in the Tree View only for visual organization of objects. Undesirable consequence: when the Electrical Device is copied, all its child CableTerminal and SuppLines objects must always be manually copied with it.
  • SuppLines or CableTerminal child objects are created automatically depending on Preset, their shape and placement are controlled by Electrical Device.

Eigenschaften

Ein Elektrisches Gerät-Objekt teilt die allgemeinen Eigenschaften und das Verhalten aller Arch Komponenten.

Daten

Component

Some most frequently used properties from the group are described here. The rest is available at Arch Component

  • DatenBase (Link): An external shape of any Shape-based base object can be used here. This will replace the default method of shape creaction and an external shape will be used. Number Of Terminals and Number Of Supp Lines parameters become visible and can be changed if Base property is not None.

Electrical Device

  • DatenNumber Of Supp Lines (Integer): (hidden by default) Specifies the number of a SuppLines child objects. The default value is 0.
  • DatenNumber Of Terminals (Integer): (hidden by default) Specifies the number of a CableTerminal child objects. The default value is 1.
  • DatenPreset (Enumeration): Specifies the predefined set of parameters for Electrical Device. The default preset is MCB_1-Pole.

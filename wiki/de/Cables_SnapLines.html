Beschreibung

NOTE: Starting with version 0.3.6, the object's name has been changed from SuppLines to SnapLines, which, in the author’s opinion, better reflects the object’s function and is hopefuly more suitable for translation. This change is backward compatible. All models created previously using the old name will continue to function properly.

The SnapLines is a dependent object and cannot be created directly.

It can be created as a child object during creation of: CableBox, CableConnector, ElectricalDevice, CableLightPoint.

It serves as a support for WireFlex objects which form a Base for Cable. The SnapLines object itself is attached to other objects mentioned above and keeps predefined placement relative to them. The SnapLines has a shape of a cross with five vertices which can be used as attachment supports for up to five cables entering single hole in a cable box. It can also be a reference point for cable entering the connector or electrical device. It is up to the user to find the SnapLines helpful or not.

Important: the real relation between SnapLines object and its parent is reversed to avoid cyclic dependencies and to speed up recalculation. This has some consequences (see Notes).

Example of a default SnapLines shape.

Usage

The SnapLines object cannot be created directly. It is made during the creation of: CableBox, CableConnector, ElectricalDevice, CableLightPoint depending on Number Of Supp Lines property of these objects.

Notes

  • The SnapLines object has reversed relation with its parent to avoid cyclic dependencies. It is shown as a child in the Tree View only for visual organization of objects. Undesirable consequence: when a parent is copied, all its child SnapLines objects must always be manually copied with it.
  • Depending of parent the single SnapLines object can have one or more crosses.

Properties

The SnapLines object has not any visible properties to edit. Its shape can be created and modified by its parent object or take the form of a cross with a default size.

Beschreibung

Viele CAM-Benutzer sind Hobbyisten oder Heimwerker. Daher verwenden sie ihre CNC-Maschinen, um selbst konfigurierten und generierten G-Code auszuführen. Bei den meisten professionellen/kommerziellen Anwendern ist dies nicht der Fall. In professionellen Werkstätten sind verschiedene Personen für die Erstellung des G-Codes (CNC-Programmierer) und für dessen Ausführung auf den Maschinen (CNC-Bediener) zuständig.

Hobbyisten führen den G-Code in der Regel nur wenige Minuten nach der Nachbearbeitung aus, wahrscheinlich nur ein- oder zweimal. In einem professionellen Betrieb kann ein bewährter G-Code nach seiner Erstellung über Monate oder Jahre hinweg viele Male ausgeführt werden.

Ein Problem, das in einer professionellen CNC-Werkstatt auftritt, ist, dass der Programmierer viele Annahmen trifft, die NICHT im G-Code selbst kommuniziert werden. Beispielsweise kann der G-Code ein Werkzeug "T3" anfordern, aber sofern dies nicht kommentiert wird, sagt der G-Code nichts darüber aus, um welche Art von Werkzeug es sich bei "T3" handelt. Es wird einfach angenommen, dass T3 im CAM-System dasselbe ist wie T3 auf der Maschine. Es gibt viele solcher Annahmen, die die Maschineneinrichtung, die Werkzeuge, das Material, die Teileausrichtung usw. betreffen. Selbst wenn der G-Code perfekt ist, kann es zu einem Absturz kommen, wenn der Bediener die Maschine nicht mit denselben Annahmen einrichtet.

Kommerzielle Werkstätten erstellen oft ein 'Einrichtungsbuch', in dem all diese Annahmen dokumentiert sind und das den Bedienern die notwendigen Informationen zur Konfiguration der Maschine und zur Herstellung eines Teils liefert.

CAM Plausibilitätsprüfung ist der Befehl im Arbeitsbereich CAM, um diese Art von Informationen zu generieren. Die Ausgabe dieses Befehls ist eine eigenständige HTML-Datei mit eingebetteten Bildern.

Oben: Beispiel eines mit der CAM Plausibilitätsprüfung erzeugten Berichts

Über den Bericht

Der Inhalt ist so weit wie möglich FreeCAD-unabhängig. Der CNC-Bediener verwendet möglicherweise niemals FreeCAD, sodass FreeCAD/CAM-spezifische Terminologie verwirrend sein kann. Der Bericht ist in verschiedene Abschnitte unterteilt und so formatiert, dass das Auffinden von Informationen einfach und vorhersehbar ist.

Information zum Teil

Dieser Abschnitt gibt einen Überblick über das, was hergestellt wird. Idealerweise sollte das Bild die Basisobjekte zeigen. Wenn es mehrere Basisobjekte gibt, sollte das Bild zeigen, wie diese ineinander verschachtelt sind.

Lauf-Zusammenfassung

Gibt einen schnellen Überblick über die minimalen und maximalen Höhen und Laufzeiten.

Rohmaterial

Beschreibt das Rohmaterialobjekt aus dem Auftrag. Dies ist ein Bereich, in dem CAM von einigen Verbesserungen profitieren würde. Eine rudimentäre Materialeigenschaft für das Rohmaterial wäre hier nützlich und könnte auch zur Empfehlung von Vorschüben/Geschwindigkeiten herangezogen werden.

Werkzeugdaten

Has subsections for each tool number used in the job. It details what the programmer assumes the tool to be and which operations are using it. This section only works with the new Toolbit system. This is another area where CAM needs improvement. Specifically, Toolbits need additional attributes about the tool like manufacturer/url/part number.

Ausgabe

Gibt Details darüber, wo und wann der G-Code nachbearbeitet wurde. Es wird auch angezeigt, ob der Auftrag optionale/obligatorische Stopps enthält, damit der Bediener weiß, ob er sich während des Betriebs sicher von der Maschine entfernen kann.

Kühlmittel

Welche Art von Kühlmittel wird gegebenenfalls benötigt?

Vorrichtungen und Werkstückspannung

Zeigt die Teile im Kontext des Lagerumschlags und zeigt auch den Teileursprung an.

Krächzen

Von CAM Plausibilitätsprüfung erkannte Warnungen und Fehler. Diese müssen nicht unbedingt Probleme darstellen, sollten jedoch zur weiteren Überprüfung notiert werden. Wenn beispielsweise dieselbe Werkzeugnummer für verschiedene Werkzeugbits verwendet wird, wird dies als Fehler angezeigt. Wenn für einen Werkzeugcontroller kein Vorschub/keine Geschwindigkeit konfiguriert ist, wird dies als Warnung angezeigt. Außerdem werden ungenutzte Werkzeugcontroller erkannt und es wird eine Warnung ausgegeben. CAM würde hier von der Möglichkeit profitieren, beliebige Hinweise oder Warnungen hinzuzufügen.

Anwendung

  1. Select a CAM Job in the Tree View.
  2. There are several ways to invoke the command:
    • Press the Sanity Check button.
    • Select the CAM → Sanity Check option from the menu.
    • Use the keyboard shortcut: P then S.
  3. The relevant information is collected into a Python dictionary and then formatted into AsciiDoc format.
  4. The AsciiDoc file is written to disk in the same location as the file that will be post-processed.
  5. An external process calls Asciidoctor to read the AsciiDoc and generate the .html.
  6. This will auto-launch the system web browser to view the generated standalone HTML report.

Hinweise

  • AsciiDoc ist ein leichtgewichtiges Auszeichnungssprachen-Format (Markup) zum Verfassen von Notizen, Artikeln, Büchern usw. Es ist für Menschen lesbar und lässt sich leicht in andere Formate übersetzen.
  • Asciidoctor ist ein schneller Open-Source-Textprozessor zur Konvertierung von AsciiDoc in HTML, PDF oder andere Formate. Er ist für Linux, Windows und macOS verfügbar. Asciidoctor wird nicht mit FreeCAD installiert. Wenn man CAM Plausibilitätsprüfung ohne Installation von Asciidoctor verwendet, wird die AsciiDoc-Quelldatei zwar generiert, aber die resultierende HTML-Datei wird nicht erstellt. Siehe Asciidoctor-Website

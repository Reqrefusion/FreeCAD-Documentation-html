Descrizione

For a finite element analysis, the geometry needs to be discretized into a FEM Mesh. This command uses Netgen (which needs to be installed on the system) to generate the mesh. Netgen meshes are not supported by Elmer.

Depending on your operating system and installation package, Netgen might be bundled with FreeCAD or not. For further information see FEM Install.

introduced in 1.0: The refactored Netgen mesher is available. It can be activated in the preferences and is installed via Python bindings as explained on the aforementioned FEM Install page.

Utilizzo

  1. Select the shape you want to analyze. For a volume, this needs to be solid or compsolid. A compsolid is necessary if your part is made from multiple materials (a compsolid can be created with the Part BooleanFragments command).
  2. There are several ways to invoke the command:
    • Press the Mesh From Shape by Netgen button.
    • Select the Mesh → Mesh From Shape by Netgen option from the menu.
  3. Optionally, set the max/min element size (the default setting usually creates meshes that are too coarse) and element order (using the Second Order checkbox).
  4. Optionally, change the Fineness to one of the predefined choices or choose UserDefined and manually edit the parameters.
  5. Click the Apply button to generate the mesh. introduced in 1.0: Optionally, use the Cancel button to abort meshing if using the new Netgen implementation.
  6. Click the OK button to generate the mesh and close the dialogue. You can also click the Cancel button to cancel the changes or creation of the mesh object.

Properties - Legacy Netgen

  • DatiMax. Size: Maximum size of the element in mm.
  • DatiMin. Size: introduced in 1.0: Minimum size of the element in mm.
  • DatiSecond order: Second order elements contain more nodes per element. Usually, it is enough to use rougher mesh to obtain same solution precision as with the first order elements,
    • true (default); second order elements,
    • false; first order elements.
  • DatiFineness: Offers predefined levels of mesh density.
  • DatiGrowth Rate: Defines how much adjacent elements can differ in size.
  • DatiNb. Segs per Edge: Defines the minimum number of mesh segments per edge.
  • DatiNb. Segs per Radius: Defines the minimum number of mesh segments per radius.
  • DatiOptimize:
    • true (default): applies optimization algorithm to improve mesh quality
    • false

Properties - Refactored Netgen

  • DatiBad Element Limit: Limit for max element angle (150-180).
  • DatiBase Element Np: If non-zero, baseelement must have BaseElementlNp points.
  • DatiBlock Fill: If enabled, block filling is performed.
  • DatiCheck Chart Boundary: If enabled, chart boundary is checked.
  • DatiCheck Impossible
  • DatiCheck Overlap: If enabled, overlapping surfaces are checked.
  • DatiCheck Overlapping Boundary: If enabled, overlapping surface mesh is checked before volume meshing.
  • DatiClose Edge Factor: Factor to restrict meshing based on close edges.
  • DatiCurvature Safety: Safety factor for curvatures (elements per radius).
  • DatiDelaunay: If enabled, Delaunay algorithm is used for 3D meshing.
  • DatiDelaunay2d: If enabled, Delaunay algorithm is used for 2D meshing.
  • DatiElement Order: High order element curvature.
  • DatiElement Size Weight: Weight of element size with respect to element shape.
  • DatiEnd Step: Last step:
    • AnalyzeGeometry
    • MeshEdges
    • MeshSurface
    • OptimizeSurface
    • MeshVolume
    • OptimizeVolume
  • DatiFill Distance: Block filling up to distance.
  • DatiFineness: Offers predefined levels of mesh density:
    • VeryCoarse
    • Coarse
    • Moderate
    • Fine
    • VeryFine
    • UserDefined
  • DatiGive Up Tolerance: Give up quality class for 3D meshing.
  • DatiGive Up Tolerance2d: Give up quality class for 2D meshing.
  • DatiGive Up Tolerance Open Quads: Give up quality class for closing open quads, greater than 100 for free pyramids.
  • DatiGlue: introduced in 1.1: Glue shapes to get a conformal mesh.
  • DatiGrowth Rate: Grading for local h.
  • DatiHeal Shape: If enabled, shape is healed before meshing.
  • DatiInvert Tets
  • DatiInvert Trigs
  • DatiLocal H: If enabled, uses local h.
  • DatiMax Outer Steps: Maximum outer steps.
  • DatiMax Size: Maximum size of the element in mm.
  • DatiMin Size: Minimum size of the element in mm.
  • DatiOnly3d Domain Nr
  • DatiOptimization Error Power: Power of error to approximate max error optimization.
  • DatiOptimization Steps2d: Number of 2D optimization steps.
  • DatiOptimization Steps3d: Number of 3D optimization steps.
  • DatiOptimize2d: 2D optimization strategy.
  • DatiOptimize3d: 3D optimization strategy.
  • DatiParallel Meshing: If enabled, uses parallel meshing.
  • DatiQuad Dominated: If enabled, uses quad dominated surface meshing.
  • DatiRelinner Safety: Radius of active environment (times h).
  • DatiSafety: Radius of local environment (times h).
  • DatiSecond Order: If enabled, uses second order element meshing.
  • DatiSecond Order Linear: If enabled, second order nodes are created by linear interpolation.
  • DatiSegments Per Edge: Minimum number of segments per edge.
  • DatiSloppy: Quality tolerances are handled less careful.
  • DatiStar Shape Class: Class starting star-shape filling.
  • DatiStart In Surface: If enabled, starts surface meshing from everywhere in surface.
  • DatiStart Step: First step:
    • AnalyzeGeometry
    • MeshEdges
    • MeshSurface
    • OptimizeSurface
    • MeshVolume
    • OptimizeVolume
  • DatiTry Hexes: If enabled, tries hexahedral elements.
  • DatiUse Local H: If enabled, uses local H.
  • DatiZRefine: introduced in 1.1: Z-refinement for extruded shapes, can create extruded hex meshes when combined with the Quad Dominated property. Will work only for first-order meshes (unless the generated elements are all hexahedrons):
    • No - Z-refinement is disabled
    • Regular - all elements have the same height
    • Custom - the height of each element must be specified
  • DatiZRefine Direction: introduced in 1.1: Z-refinement direction (change if the shape is extruded in a direction other than the Z-axis).
  • DatiZRefine Size: introduced in 1.1: Z-refinement size (element height) given as a fraction of the shape size (total height). For a regular partition, only one value is needed.

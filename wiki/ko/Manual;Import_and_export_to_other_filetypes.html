FreeCAD는 다양한 파일 형식으로 가져오기(import)와 내보내기(export)를 지원합니다. 다음은 사용 가능한 주요 파일 형식과 해당 기능에 대한 간단한 설명 목록입니다:

포멧 가져오기 내보내기 비고
STEP 가능 가능 이 형식은 솔리드 형상과 NURBS를 모두 지원하므로, 현재 사용 가능한 가져오기/내보내기 형식 중 가장 충실도가 높은 형식입니다. 가능한 경우에는 항상 이 형식을 사용하는 것이 좋습니다.
IGES 가능 가능 비교적 오래된 솔리드 형식이지만, 여전히 매우 잘 지원됩니다.

일부 오래된 응용 프로그램은 STEP 형식은 지원하지 않지만 IGES는 지원하는 경우가 있습니다.

BREP 가능 가능 FreeCAD의 기하 커널인 OpenCascade의 기본(native) 파일 형식입니다.
DXF 가능 가능 Autodesk에서 유지·관리하는 개방형 형식입니다. 하지만 DXF 파일 내부의 3D 데이터는 독점(proprietary) 형식으로 인코딩되어 있기 때문에, FreeCAD에서는 이 형식을 통해 2D 데이터만 가져오기/내보내기 할 수 있습니다.
DWG 가능 가능 DXF의 독점(proprietary) 버전입니다. 이 형식을 사용하려면 Teigha File Converter 유틸리티를 설치해야 합니다. 또한 이 형식은 DXF와 동일한 제한 사항을 가지고 있습니다.
OBJ 가능 가능 메시(mesh) 기반 형식입니다. 이 형식은 삼각형으로 분할된 메시만 포함할 수 있습니다. 따라서 FreeCAD의 모든 솔리드 및 NURBS 기반 객체는 내보내기 시 메시로 변환됩니다. 또한 Arc 작업대(workbench)에서는 건축 모델 내보내기에 더 적합한 대체 내보내기(exporter)*를 제공합니다.
DAE 가능 가능 SketchUp의 주요 가져오기/내보내기 형식입니다. 이 형식은 삼각형으로 분할된 메시만 포함할 수 있으며, FreeCAD의 모든 솔리드 및 NURBS 기반 객체는 내보내기 시 메시로 변환됩니다.
STL 가능 가능 메시(mesh) 기반 형식으로, 3D 프린팅에 일반적으로 사용됩니다. 이 형식은 삼각형으로 분할된 메시만 포함할 수 있으며, FreeCAD의 모든 솔리드 및 NURBS 기반 객체는 내보내기 시 메시로 변환됩니다.
PLY 가능 가능 오래된 메시(mesh) 기반 형식입니다. 이 형식은 삼각형으로 분할된 메시만 포함할 수 있으며, FreeCAD의 모든 솔리드 및 NURBS 기반 객체는 내보내기 시 메시로 변환됩니다.
IFC 가능 가능 Industry Foundation Classes 형식입니다. 이 형식을 사용하려면 IfcOpenShell-python의 설치가 필요합니다. IFC 형식은 다른 응용 프로그램과의 호환성이 복잡한 편이므로, 사용 시 주의가 필요합니다.
SVG 가능 가능 우수하고 널리 사용되는 2D 그래픽 형식입니다.
VRML 가능 가능 다소 오래된 웹용 메시(mesh) 기반 형식입니다.
GCODE 가능 가능 FreeCAD는 이미 여러 종류의 GCode 형식에 대해 가져오기 및 내보내기를 지원하지만, 현재로서는 지원되는 기계의 수가 많지 않습니다.
CSG 가능 불가능 OpenSCAD의 CSG(구성 솔리드 기하, Constructive Solid Geometry) 형식입니다.

이들 파일 형식 중 일부에는 옵션 설정이 존재합니다. 이 옵션들은 메뉴 편집(Edit) → 기본 설정(Preferences) → 가져오기/내보내기(Import/Export):에서 설정할 수 있습니다.

Description

Câble crée un objet Cable basé sur des Cables WireFlex et Cables Profils sélectionnés. Le composé peut également être sélectionné comme polyligne de base (introduit dans la version 0.2.0). Le câble est entièrement paramétrique et est dérivé de Arch Conduite et hérite de toutes ses propriétés. Certaines propriétés spécifiques à Arch Conduite sont cachées et ne sont pas utilisées par Câble.

Le câble crée automatiquement plusieurs sous-composants sur la base d'un profil donné. Ces sous-composants sont les suivants :

  • Deux clones de profils (A_Profile et B_Profile) positionnés aux deux extrémités d'un fil de Base et alignés perpendiculairement aux extrémités de la polyligne Base.
  • Plusieurs fils secondaires WireFlex supplémentaires attachés aux profils A et B aux points centraux des fils monoconducteurs. Les fils secondaires WireFlex sont nommés en fonction de la propriété Sub Colors.

L'objet Cable peut être coloré avec l'outil Arch Multi-matériaux rattaché à la propriété Material.

Utilisation

  1. Sélectionnez le WireFlex ou le composé existant dans la vue 3D.
  2. Vous pouvez également sélectionner le profil du câble existant dans la vue 3D ou la vue en arborescence en maintenant la touche CTRL.
    Si le profil n'est pas sélectionné, un câble constitué d'une seule polyligne sera créé.
  3. Créez le câble par l'une des méthodes suivantes :
    • Appuyez sur le bouton Cable.
    • Sélectionnez l'option Cables → Cable du menu.
    • Cliquez avec le bouton droit de la souris dans la vue en arborescence ou la vue 3D et sélectionnez l'option Cables → Cable dans le menu contextuel.

Une fois créé, le câble est placé à l'emplacement du WireFlex sélectionné dans l'espace 3D.

Remarques

  • Tous les sous-composants du câble (polyligne de base, sous-profils, sous-polylignes) sont visibles par défaut. Ils peuvent être facilement cachés en modifiant la propriété Show Sub Lines d'un câble.
  • Comme première étape dans Utilisation, au lieu de sélectionner un seul WireFlex, une série de plusieurs objets WireFlex et/ou CompoundPath peut être sélectionnée. Un nouveau CompoundPath est alors créé à partir de la sélection et utilisé comme base pour le câble (introduit dans la version 0.2.0).
  • La première arête de chaque sous-polyligne a toujours la longueur = 5mm (codé en dur et ne peut être modifié par l'utilisateur) et est toujours perpendiculaire au sous-profil attaché. Ceci est nécessaire pour assurer une création correcte de la forme de la sous-polyligne.
  • Les éventuels congés ne sont pas appliqués par défaut. Ils peuvent être ajustés en modifiant les propriétés du câble : Base Wire Fillet Radius et Sub Wires Fillet Radius. Il est recommandé d'effectuer cette opération à la fin de l'édition d'un câble.
  • Le profil est principalement utilisé pour créer des câbles à 2 polylignes et plus. Bien qu'il soit possible de créer un câble mono-polyligne à double isolation en utilisant le profil, la méthode la plus simple pour créer un câble mono-polyligne est de ne pas utiliser de profil du tout.
  • Dans de rares cas, il peut arriver que le deuxième profil (B_Profile) d'un câble soit dans un état d'erreur après quelques changements de paramètres. Dans ce cas, le Map Mode de B_Profile doit être modifié en Deactivated et recalculé. Cela résoudra le problème.

Propriétés

Un objet Cable partage les propriétés communes et les comportements de tous les Arch Composants.

Un câble est dérivé de Arch Conduite et hérite de toutes ses propriétés. Certaines propriétés spécifiques à Arch Conduite sont cachées et ne sont pas utilisées par l'objet Cable.

Données

Component

  • DonnéesBase (Link) : la polyligne de base d'un câble.

Pour les autres propriétés du groupe, voir Arch Composant.

Cache (Structure introduite dans la version 0.2.0)

  • DonnéesAuto Label Sub Lines (Bool) : si la valeur est true, les noms de tous les sous-éléments (fil de base, sous-profils, sous-fils) seront renommés automatiquement. Pour rafraîchir les noms, la propriété doit être changée en false et en true à nouveau. Les extrémités du câble sont nommées A et B. Les suffixes des noms des sous-câbles sont nommés d'après les noms de la propriété Sub Colors.
  • DonnéesCable Rotation (Angle) : spécifie l'angle de rotation du sous-profil A. La rotation du sous-profil B est calculée automatiquement. Ce paramètre est utile pour les câbles plats.
  • DonnéesProfile (Link) : profil de base de ce câble. Il détermine le nombre de polylignes dans le câble. S'il n'est pas défini, un câble mono-polyligne est créé.
  • DonnéesShow Sub Lines (Bool) : si la valeur est true, tous les sous-éléments (polyligne de base, profils, sous-lignes) sont visibles dans la vue 3D.
  • DonnéesSub Colors (StringList) : utilisé comme palette de couleurs pour la propriété Material et comme source de noms pour la propriété Auto Label Sub Lines. Il contient la liste des noms avec la formule suivante : "Label:nr", où Label indique le nom de la gaine du câble, de l'isolation du sous-fil ou du conducteur (exemples : J, L1, N, PE, CU) et nr indique le numéro du sous-fil (numéros spéciaux : 0 - gaine de câble, -1 - tous les autres, utilisés principalement pour l'étiquette CU). Pour les câbles créés avec 1-5 ou 8 sous-fils, les valeurs par défaut sont placées ici. La propriété peut être modifiée en cliquant sur ... à côté du nom de la propriété.
  • DonnéesSub Profiles (LinkList) : liste des liens vers les sous-profils A et B. Ils sont créés automatiquement en tant que clones du profil principal lors de la création du câble.
  • DonnéesSub Wires (LinkList) : liste des liens vers tous les sous-fils aux deux extrémités d'un câble (A et B). Tous les fils secondaires sont créés automatiquement lors de la création du câble.

Cable Shape (Structure introduite dans la version 0.2.0)

  • DonnéesBase Wire Fillet Radius (Length) : spécifie le rayon des congés aux coins du fil de base.
  • DonnéesBase Wire Path Type (Enumeration) : Spécifie le type de chemin du fil de base. Par défaut = "Wire".
  • DonnéesLength (Length) : affiche la longueur totale d'un câble ainsi que les extrémités des fils secondaires.
  • DonnéesStripped Wire Length (Length) : longueur du fil conducteur dénudé à l'extrémité de chaque sous-fil. Par défaut = 8 mm.
  • DonnéesSub Wires Fillet Radius (Length) : spécifie le rayon des congés aux coins de tous les fils secondaires.
  • DonnéesSub Wires Path Type (Enumeration) : spécifie le type de chemin de tous les fils secondaires. Si la valeur "Customized" est sélectionnée, les différentes valeurs des fils secondaires ont été définies indépendamment. Par défaut = "Wire".

Flux de travail typique

Créer un câble coloré complet nécessite quelques étapes :

  1. Créez le câble comme décrit dans Utilisation.
  2. Renommez le câble comme vous le souhaitez. Si la propriété Auto Label Sub Lines est activée, tous les sous-composants du câble seront renommés en conséquence. Les suffixes des noms des sous-câbles suivent les étiquettes de couleur dans la propriété Sub Colors. Ces étiquettes peuvent également être modifiées à volonté.
  3. Créez un ensemble de multimatériaux par défaut comme décrit dans Cables Matériau. Ajustez les matériaux créés ou laissez les valeurs par défaut.
  4. Attribuez le multimatériau sélectionné à la propriété Material d'un câble.
  5. Assurez-vous que les étiquettes de couleur dans la propriété Sub Colors correspondent aux noms dans le multimatériau choisi. Cela permet de garantir des couleurs correctes pour le câble et tous ses sous-fils.
  6. Chaque sous-câble peut être modifié comme n'importe quel WireFlex et peut être connecté à d'autres éléments, par exemple à un connecteur. Remarque : la première arête de chaque sous-fil a toujours une longueur de 5 mm (codée en dur et ne pouvant être modifiée par l'utilisateur) et est toujours perpendiculaire au sous-profil attaché. Ceci est nécessaire pour assurer une création correcte de la forme du sous-fil.

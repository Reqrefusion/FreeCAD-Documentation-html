Description

Pour une analyse par éléments finis, la géométrie doit être discrétisée en FEM Maillage. Cette commande utilise Netgen (à installer sur le système) pour générer le maillage. Les maillages Netgen ne sont pas pris en charge par le solveur Elmer.

En fonction de votre système d'exploitation et de le paquetage d'installation, Netgen peut être fourni avec FreeCAD ou pas. Pour plus d'informations voir FEM Installation des composants requis.

introduced in 1.0: The refactored Netgen mesher is available. It can be activated in the preferences and is installed via Python bindings as explained on the aforementioned FEM Install page.

Utilisation

  1. Sélectionnez la forme que vous voulez analyser. Pour un volume, il doit s'agir d'un solide ou d'un compsolide (composé de solides). Un compsolid est nécessaire si votre pièce est faite de plusieurs matériaux. (Un compsolid peut être créé avec la commande Part Fragments booléens).
    1. Il y a plusieurs façons de lancer la commande :
    • Appuyez sur le bouton Mailler avec le mailleur Netgen
    • Sélectionnez le bouton Maillage → Mailler avec le mailleur Netgen du menu.
  2. Vous pouvez également définir la taille max/min des éléments (le paramètre par défaut crée généralement des maillages trop grossiers) et l'ordre des éléments (en utilisant la case à cocher Second ordre).
  3. Vous pouvez également changer la Précision pour l'un des choix prédéfinis ou choisir Défini par l'utilisateur et éditer manuellement les paramètres.
  4. Cliquez sur le bouton Appliquer pour générer le maillage. introduit dans la version 1.0 : vous pouvez utiliser le bouton Annuler pour annuler le maillage si vous utilisez la nouvelle implémentation de Netgen.
  5. Cliquez sur le bouton OK pour générer le maillage et fermer la fenêtre de dialogue. Vous pouvez également cliquer sur le bouton Annuler pour annuler les modifications ou la création de l'objet de maillage.

Propriétés

  • DonnéesMax. Size : taille maximale de l'élément en mm.
  • DonnéesMin. Size : taille minimale de l'élément en mm. introduit dans la version 1.0
  • DonnéesSecond order : les éléments de second ordre contiennent plus de noeuds par élément. En général, il suffit d'utiliser un maillage plus grossier pour obtenir la même précision de solution qu'avec les éléments de premier ordre,
    • true (valueur par défaut) ; éléments de second ordre,
    • false ; éléments de premier ordre.
  • DonnéesFineness : offre des niveaux prédéfinis de densité de maillage.
  • DonnéesGrowth Rate : définit de combien les éléments adjacents peuvent différer en taille.
  • DonnéesNb. Segs per Edge : définit le nombre minimum de segments de maille par arête.
  • DonnéesNb. Segs per Radius : définit le nombre minimum de segments de maillage par rayon.
  • DonnéesOptimize :
    • true (valeur par défaut) : applique un algorithme d'optimisation pour améliorer la qualité du maillage
    • false

Properties - Refactored Netgen

  • DonnéesBad Element Limit: Limit for max element angle (150-180).
  • DonnéesBase Element Np: If non-zero, baseelement must have BaseElementlNp points.
  • DonnéesBlock Fill: If enabled, block filling is performed.
  • DonnéesCheck Chart Boundary: If enabled, chart boundary is checked.
  • DonnéesCheck Impossible
  • DonnéesCheck Overlap: If enabled, overlapping surfaces are checked.
  • DonnéesCheck Overlapping Boundary: If enabled, overlapping surface mesh is checked before volume meshing.
  • DonnéesClose Edge Factor: Factor to restrict meshing based on close edges.
  • DonnéesCurvature Safety: Safety factor for curvatures (elements per radius).
  • DonnéesDelaunay: If enabled, Delaunay algorithm is used for 3D meshing.
  • DonnéesDelaunay2d: If enabled, Delaunay algorithm is used for 2D meshing.
  • DonnéesElement Order: High order element curvature.
  • DonnéesElement Size Weight: Weight of element size with respect to element shape.
  • DonnéesEnd Step: Last step:
    • AnalyzeGeometry
    • MeshEdges
    • MeshSurface
    • OptimizeSurface
    • MeshVolume
    • OptimizeVolume
  • DonnéesFill Distance: Block filling up to distance.
  • DonnéesFineness: Offers predefined levels of mesh density:
    • VeryCoarse
    • Coarse
    • Moderate
    • Fine
    • VeryFine
    • UserDefined
  • DonnéesGive Up Tolerance: Give up quality class for 3D meshing.
  • DonnéesGive Up Tolerance2d: Give up quality class for 2D meshing.
  • DonnéesGive Up Tolerance Open Quads: Give up quality class for closing open quads, greater than 100 for free pyramids.
  • DonnéesGlue: introduced in 1.1: Glue shapes to get a conformal mesh.
  • DonnéesGrowth Rate: Grading for local h.
  • DonnéesHeal Shape: If enabled, shape is healed before meshing.
  • DonnéesInvert Tets
  • DonnéesInvert Trigs
  • DonnéesLocal H: If enabled, uses local h.
  • DonnéesMax Outer Steps: Maximum outer steps.
  • DonnéesMax Size: Maximum size of the element in mm.
  • DonnéesMin Size: Minimum size of the element in mm.
  • DonnéesOnly3d Domain Nr
  • DonnéesOptimization Error Power: Power of error to approximate max error optimization.
  • DonnéesOptimization Steps2d: Number of 2D optimization steps.
  • DonnéesOptimization Steps3d: Number of 3D optimization steps.
  • DonnéesOptimize2d: 2D optimization strategy.
  • DonnéesOptimize3d: 3D optimization strategy.
  • DonnéesParallel Meshing: If enabled, uses parallel meshing.
  • DonnéesQuad Dominated: If enabled, uses quad dominated surface meshing.
  • DonnéesRelinner Safety: Radius of active environment (times h).
  • DonnéesSafety: Radius of local environment (times h).
  • DonnéesSecond Order: If enabled, uses second order element meshing.
  • DonnéesSecond Order Linear: If enabled, second order nodes are created by linear interpolation.
  • DonnéesSegments Per Edge: Minimum number of segments per edge.
  • DonnéesSloppy: Quality tolerances are handled less careful.
  • DonnéesStar Shape Class: Class starting star-shape filling.
  • DonnéesStart In Surface: If enabled, starts surface meshing from everywhere in surface.
  • DonnéesStart Step: First step:
    • AnalyzeGeometry
    • MeshEdges
    • MeshSurface
    • OptimizeSurface
    • MeshVolume
    • OptimizeVolume
  • DonnéesTry Hexes: If enabled, tries hexahedral elements.
  • DonnéesUse Local H: If enabled, uses local H.
  • DonnéesZRefine: introduced in 1.1: Z-refinement for extruded shapes, can create extruded hex meshes when combined with the Quad Dominated property. Will work only for first-order meshes (unless the generated elements are all hexahedrons):
    • No - Z-refinement is disabled
    • Regular - all elements have the same height
    • Custom - the height of each element must be specified
  • DonnéesZRefine Direction: introduced in 1.1: Z-refinement direction (change if the shape is extruded in a direction other than the Z axis).
  • DonnéesZRefine Size: introduced in 1.1: Z-refinement size (element height) given as a fraction of the shape size (total height). For a regular partition, only one value is needed.

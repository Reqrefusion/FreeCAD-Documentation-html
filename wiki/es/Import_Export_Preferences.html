Introducción

FreeCAD puede importar y exportar muchos formatos de archivo. Para algunos formatos existen preferencias específicas. Estas se encuentran en el Editor de preferencias. En el menú, seleccione 'Editar → Preferencias...' y luego 'Importar-Exportar'.

No todas las páginas de preferencias de importación y exportación están disponibles por defecto. Para algunas, es necesario cargar primero un entorno de trabajo.

Notas

Páginas de TechDraw

Las preferencias DXF y SVG que se enumeran aquí no la utilizan los comandos Exportar página como SVG y Exportar página como DXF del Entorno de Trabajo TechDraw, ni por la opción de exportación TechDraw: 'Archivo → Exportar → Dibujo técnico (*.svg *.dxf *.pdf)'.

Archivos IFC

Algunas preferencias de importación y exportación de IFC específicas de NativeIFC se pueden encontrar en Preferencias de BIM.

Archivos OpenSCAD

Las preferencias de importación y exportación de archivos OpenSCAD se pueden encontrar en Preferencias de OpenSCAD.

Nota para usuarios de Linux: Para manejar este formato de archivo, FreeCAD requiere el módulo pyCollada.

En esta página puede especificar lo siguiente:

Mallador Establece el programa de mallado que se debe usar. Si usa Netgen, asegúrese de que esté disponible. Esto se puede comprobar usando Entorno de Trabajo de Mallado y creando una malla con Netgen. Si no está disponible, debe instalarse otra versión de FreeCAD compilada con Netgen.
Teselación Valor de teselación que se utilizará con los programas de mallado «Builtin» y «Mefisto».
Gradación Valor de gradación que se utilizará para el mallado con «Netgen». Este valor describe la rapidez con la que disminuye el tamaño de la malla. El gradiente del tamaño de la malla local h(x) está limitado por abs(Δh(x)) ≤ 1/value.
Segmentos por arista Número máximo de segmentos por arista.
Segments por radio Número de segmentos por radio.
Segundo orden Permite una malla de segundo orden.

DWG

DWG (del dibujo) es un formato de archivo binario propietario y de código cerrado para almacenar datos y metadatos de diseño 2D y 3D. FreeCAD requiere convertidores externos para procesar archivos DWG.

Nota: Todas las configuraciones para el formato de archivo DXF también se aplican a DWG.

En esta página puede especificar lo siguiente:

Nombre Descripción
Método de conversión Seleccione el convertidor DWG que desea usar:
  • Automático: FreeCAD intentará encontrar un convertidor automáticamente siguiendo el orden del resto de esta lista. introduced in 0.21: Para LibreDWG, se busca en la ruta de búsqueda del sistema operativo (os.getenv("RUTA")); para los demás convertidores. Se asumen las rutas de instalación predeterminadas.
  • LibreDWG: LibreDWG es una biblioteca de código abierto para leer y escribir archivos DWG. Carece de compatibilidad con varias entidades DWG y puede que no siempre ofrezca resultados precisos.
  • Convertidor ODA: El Convertidor de archivos ODA es una utilidad gratuita proporcionada por Open Design Alliance. Ofrece resultados muy buenos y fiables.
  • QCAD pro: QCAD pro es la versión de pago de la plataforma CAD 2D de código abierto QCAD basada en DXF. Su convertidor DWG utiliza las bibliotecas Teigha de OpenDesign Alliance y, por lo tanto, ofrece los mismos buenos resultados que el Convertidor de archivos ODA.
Ruta al convertidor de archivos Si FreeCAD no encuentra un convertidor, debe especificar una ruta aquí:
  • LibreDWG: dwg2dxf o dxf2dwg en Linux y macOS, dwg2dxf.exe o dxf2dwg.exe en Windows. Se puede especificar cualquiera de los dos nombres de archivo tanto para la conversión de DWG como para la conversión a DWG.
  • Convertidor ODA: ODAFileConverter en Linux y macOS, ODAFileConverter.exe en Windows.
  • QCAD pro: dwg2dwg (un script bash) en Linux y macOS, dwg2dwg.bat en Windows.

introduced in 0.21: La ruta debe contener el nombre de archivo correcto.

DXF

AutoCAD DXF (Drawing eXchange Format) es un formato propietario para intercambiar datos CAD entre AutoCAD y otros programas.

En esta página puede especificar lo siguiente:

Nombre Descripción
Mostrar este diálogo al importar y exportar Si está marcada, este diálogo de preferencias se mostrará al importar o exportar archivos DXF.
Usar el importador de Python heredado Si está marcada, se utiliza el importador de Python; de lo contrario, se utiliza el importador de C++ más reciente. El importador de C++ es más rápido, pero aún no tiene tantas funciones.

El importador de Python utiliza la preferencia 'Editar → Preferencias... → Borrador → Configuración general → Nivel de precisión interna'. Para obtener una importación precisa, establezca este valor en 8 o superior.

Usar exportador de Python heredado Si está marcada, se utiliza el exportador de Python; de lo contrario, se utiliza el más reciente de C++. El exportador de C++ es más rápido, pero aún no cuenta con tantas funciones.
Permitir que FreeCAD descargue y actualice automáticamente las bibliotecas DXF Al marcar esta opción, permitirá que FreeCAD descargue el Convertidor de Python para la importación y exportación de DXF. Este convertidor no se puede incluir con FreeCAD porque tiene una licencia de software diferente.
Importar Seleccione qué se importará.

Si textos y dimensiones está marcada, los textos y mtexts también se importarán.

Si puntos está marcada, los puntos también se importarán.

Si layouts está marcada, también se importarán los objetos del espacio papel.

Si *blocks está marcada, también se importarán los bloques anónimos (cuyos nombres comienzan con *).

Crear Seleccione lo que se creará.

Si se selecciona Formas de piezas simples, solo se crearán objetos de piezas estándar. Esta es la opción más rápida.

Si se selecciona Objetos de borrador, se crearán objetos de borrador paramétricos siempre que sea posible.

Si se selecciona Bocetos, se crearán bocetos siempre que sea posible.

Factor de escala para aplicar a los archivos importados Factor de escala que se aplicará a los archivos DXF al importarlos. El factor es la conversión entre las unidades del archivo DXF y los milímetros. Ejemplo: para archivos en milímetros: 1, en centímetros: 10, en metros: 1000, en pulgadas: 25,4, en pies: 304,8.
Usar colores del archivo DXF Si está marcada, se recuperarán los colores de los objetos DXF siempre que sea posible. De lo contrario, se aplicarán los colores predeterminados.
Unir geometría Si está marcada, FreeCAD intentará unir los objetos coincidentes en líneas. ¡Tenga en cuenta que esto puede tardar un tiempo!
Agrupar capas en bloques Si está marcada, los objetos de las mismas capas se unirán en bloques de borrador, que se muestran más rápido, pero son más difíciles de editar.
Usar tamaño de fuente estándar para los textos Si está marcada, los textos importados tendrán el tamaño estándar Texto de borrador, en lugar del tamaño que tienen en el documento DXF.
Usar capas Si está marcada, las capas DXF se importarán como Capas de borrador.
Importar contornos de sombreado como alambres Si está marcada, los sombreados se convertirán en alambres simples.
Renderizar polilíneas con ancho Si está marcada, si las polilíneas tienen un ancho definido, se representarán como líneas cerradas con el ancho correcto.
Tratar elipses y splines como polilíneas La exportación de elipses y splines no es totalmente compatible. Utilice esta opción para exportarlas como polilíneas.

La opción Max Spline Segment define la longitud máxima de cada segmento de polilínea. Si se establece en 0, toda la spline se trata como un segmento recto.

Exportar objetos 3D como mallas de policaras Si está marcada, todos los objetos que contengan caras se exportarán como policaras 3D.
Exportar vistas de TechDraw como bloques Si está marcada, las vistas de TechDraw se exportarán como bloques. Esto podría fallar con plantillas posteriores a DXF R12.
Proyectar objetos exportados en la dirección de la vista actual Si está marcada, los objetos exportados se proyectarán para reflejar la dirección de la vista actual. Esta opción solo funciona si se utiliza el exportador de Python heredado.

IFC

Industry Foundation Classes (IFC) es un formato ampliamente utilizado para intercambiar datos entre aplicaciones BIM. Se utiliza en arquitectura e ingeniería.

Nota para usuarios de Linux: Para manejar este formato de archivo, FreeCAD requiere el módulo IfcOpenShell.

Importación IFC

En esta página puede especificar lo siguiente:

Nombre Descripción
Mostrar este diálogo al importar Si está marcada, este diálogo de preferencias se mostrará al importar archivos IFC.
Mostrar mensajes de depuración Muestra mensajes de depuración detallados durante la importación y exportación de archivos IFC en la Vista de informe.
Crear clones cuando los objetos comparten geometría Los objetos IFC pueden compartir la misma definición de geometría entre varios objetos; solo su ubicación es diferente. Cuando esta opción está habilitada, se utilizan clones para lograr el mismo resultado en FreeCAD. Un objeto es el objeto base, los demás son clones.
Número de núcleos a usar (experimental) Especifique el número de núcleos de CPU que se utilizarán para la importación de IFC. El número máximo debe ser menor que el número de núcleos disponibles. Use 0 para desactivar esta función.
Importar objetos IFC de arquitectura como Lo que se creará en FreeCAD para los objetos IFC de arquitectura.
Importar objetos IFC de estructura como Lo que se creará en FreeCAD para los objetos IFC de estructura.
Elemento raíz Solo se importarán los subtipos del elemento especificado. Mantenga el elemento predefinido IfcProduct para importar todos los elementos de construcción.
Separar aberturas Si está marcada, las aberturas se importarán como sustracciones; de lo contrario, las formas de los muros ya tendrán sus aberturas sustraídas.
Detectar extrusiones Si está marcada, el importador intentará detectar extrusiones. Tenga en cuenta que esto podría ralentizar el proceso.
Dividir muros multicapa Divide los muros compuestos en varias capas.
Añadir prefijo a los nombres con el número de ID Si está marcada, los nombres de los objetos se antepondrán al número de ID IFC [1].
Combinar materiales con el mismo nombre y color Si se encuentran varios materiales con el mismo nombre en el archivo IFC, se tratarán como uno solo.
Importar propiedades IFC en una hoja de cálculo Si está marcada, cada objeto tendrá sus propiedades IFC [2] almacenadas en una hoja de cálculo.
Permitir formas no válidas Si no está marcada, no se importarán las formas no válidas.
Excluir lista Lista separada por comas de entidades IFC [3] que se excluirán de las importaciones.
Ajustar vista durante la importación Ajusta la vista durante la importación de los objetos importados. Esto ralentizará la importación, pero permite supervisarla.
Importar definiciones paramétricas completas de FreeCAD si están disponibles Crea un modelo paramétrico completo durante la importación utilizando las propiedades de los objetos de FreeCAD almacenadas.

Para obtener las propiedades de FreeCAD, el modelo debe haberse exportado utilizando la opción Exportar modelo paramétrico completo de FreeCAD.

Reemplazar 'Proyecto', 'Sitio', 'Edificio' y 'Planta' por 'Grupo' Si está marcada, se utilizarán grupos para reemplazar los objetos mencionados.

Exportación IFC

En esta página puede especificar lo siguiente:

Nombre Descripción
Mostrar este diálogo al exportar Si está marcada, este diálogo de preferencias se mostrará al exportar archivos IFC.
Tipo de exportación Seleccione cómo se debe exportar el modelo: como Modelo estándar, Análisis estructural o Estándar + estructural.
Forzar exportación como BREP Algunos visores de IFC no admiten objetos exportados como extrusiones. Use esta opción para forzar la exportación de todos los objetos como geometría BREP. Pero evite exportar como Brep si es posible, ya que esto convierte los objetos en no paramétricos.
Usar opciones de triangulación DAE Usar las opciones de triangulación configuradas en la página de opciones DAE.
Unir facetas coplanares al triangular Las formas curvas que no se pueden representar como curvas en IFC se descomponen en facetas planas. Si esta opción está marcada, se realiza un cálculo adicional para unir las facetas coplanares.
Almacenar ID único de IFC en objetos de FreeCAD Al exportar objetos sin un ID único (UID), el UID generado se almacenará dentro del objeto de FreeCAD para su reutilización la próxima vez que se exporte. Esto reduce las diferencias entre las versiones de los archivos.
Usar el serializador IfcOpenShell si está disponible IFCOpenShell es una biblioteca para archivos IFC. Su funcionalidad de serialización puede generar geometría IFC válida a partir de formas OCC.

Tenga en cuenta que esta función aún está en fase experimental.

Exportar objetos 2D como IfcAnnotations Si está marcada, los objetos 2D se exportarán como IfcAnnotation.
Exportar modelo paramétrico completo de FreeCAD Si está marcada, todas las propiedades de los objetos de FreeCAD se almacenarán dentro de los objetos exportados, lo que permite recrear un modelo paramétrico completo al reimportarlo mediante la opción Importar definiciones paramétricas completas de FreeCAD si están disponibles.
Reutilizar entidades similares Si está marcada, las entidades similares se utilizarán solo una vez en el archivo, si es posible. Esto puede reducir considerablemente el tamaño del archivo, pero dificultará su lectura.
Deshabilitar IfcRectangleProfileDef Siempre que sea posible, los objetos IFC que sean rectángulos extruidos se exportarán como IfcRectangleProfileDef. Para las aplicaciones que tengan problemas para importar estas entidades, seleccione esta opción para asegurarse de que todos los perfiles se exporten como IfcArbitraryClosedProfileDef.
Detección automática y exportación como casos estándar cuando corresponda Algunos tipos de IFC como IfcWall o IfcBeam tienen versiones estándar especiales como IfcWallStandardCase ​​o IfcBeamStandardCase. Si se selecciona esta opción, FreeCAD exportará automáticamente dichos objetos como casos estándar cuando se cumplan las condiciones necesarias.
Añadir emplazamiento predeterminado si no se encuentra en el documento Al exportar un archivo IFC, si no se encuentra ningún emplazamiento en el documento de FreeCAD, se añade uno predeterminado. Un emplazamiento no es obligatorio según el estándar IFC, pero es práctica común incluir al menos uno en el archivo.
Añadir planta predeterminada si no se encuentra en el documento Al exportar un archivo IFC, si no se encuentra ninguna planta en el documento de FreeCAD, se añade una predeterminada. Una planta no es obligatoria según el estándar IFC, pero es práctica común incluir al menos una en el archivo.
Unidades del archivo IFC Seleccione las unidades que se utilizarán al exportar archivos IFC.
Añadir edificio predeterminado si no se encuentra ninguno en el documento Al exportar un archivo IFC, si no se encuentra ningún edificio en el documento de FreeCAD, se añade uno predeterminado.

Advertencia: El estándar IFC requiere al menos un edificio en cada archivo. Si desactiva esta opción, generará un archivo IFC no estándar.

Sin embargo, en FreeCAD creemos que tener un edificio no debería ser obligatorio, y esta opción está disponible para que podamos demostrar nuestro punto de vista.

En FreeCAD, es posible anidar grupos dentro de edificios o plantas. Si esta opción está desactivada, los grupos de FreeCAD se guardarán como IfcGroups y se agregarán a la estructura del edificio. Sin embargo, los estándares IFC no recomiendan agregar elementos que no sean edificios, como IfcGroups. Por lo tanto, también es posible exportar estos grupos como IfcElementAssemblies, lo que genera un archivo compatible con IFC. No obstante, en FreeCAD creemos que debería ser posible anidar grupos dentro de estructuras, y esta opción está disponible para que podamos demostrar nuestro punto de vista.

IGES

El formato de archivo Initial Graphics Exchange Specification (IGES) permite el intercambio digital de información entre sistemas CAD. Tras la publicación del formato de archivo STEP, el desarrollo de IGES se detuvo en 1996, pero aún es compatible con muchos programas CAD. Los archivos IGES tienen la extensión .iges o .igs.

Esta página solo se muestra si Part Workbench, PartDesign Workbench o OpenSCAD Workbench se ha cargado en la sesión actual de FreeCAD.

En esta página puede especificar lo siguiente:

Nombre Descripción
Unidades para la exportación de IGES Seleccione la unidad que se utilizará al exportar archivos IGES.
Escribir sólidos y superficies como Seleccione cómo se deben generar los sólidos y las superficies.

Si se selecciona Grupos de superficies recortadas (tipo 144), se exportarán como superficies recortadas.

Si se selecciona Sólidos (tipo 186) y cáscaras (tipo 514) / Modo B-REP, los sólidos se exportarán como objetos B-Rep de sólidos de tipo manifold y las cáscaras como cáscaras.

Omitir entidades en blanco Si está marcada, no se importarán las entidades en blanco [4].
Empresa Si no está vacía, el texto introducido se utilizará en el encabezado del archivo IGES para la empresa.
Autor Si no está vacío, el texto introducido se utilizará en el encabezado del archivo IGES para el autor.
Producto Si no está vacío, el texto introducido se utilizará en el encabezado del archivo IGES para el producto.

INP

INP es el formato de archivo de entrada para el software FEM Abaqus. Se utiliza para el solucionador CalculiX en el FEM Workbench.

Esta página solo se muestra si el FEM Workbench se ha cargado en la sesión actual de FreeCAD.

En esta página puede especificar lo siguiente:

Nombre Descripción
Qué elementos de malla exportar Seleccione qué elementos de malla se deben exportar.

Si se selecciona Todos, se exportarán todos los elementos.

Si se selecciona Máximo, solo se exportarán los elementos de mayor altura. Esto significa volúmenes para una malla volumétrica y caras para una malla de superficie.

Exportar datos de grupo Si está marcada, también se exportarán los grupos de malla. Cada restricción y, si hay diferentes materiales, cada material consta de dos grupos de malla: caras y nodos donde se aplica la restricción o el material.

STEP

The Standard for The Exchange of Product model data (STEP) file format is an ISO standard for the computer-interpretable representation and exchange of product manufacturing information. STEP is commonly used to exchange 3D data between CAD software. STEP files have the .step or .stp extension. For compressed files the .stpz extension is used.

This page is only shown if the Part Workbench, PartDesign Workbench, or OpenSCAD Workbench has been loaded in the current FreeCAD session.

Export invisible objects Uncheck this to skip invisible object when exporting, which is useful for CAD applications that do not support invisibility STEP styling.
Export single object placement Check this option to keep the placement information when exporting a single object. Please note that when re-importing the STEP file, the placement will be encoded into the shape geometry, instead of keeping it inside the Placement property.
Use legacy export function Use the legacy export function.

Scheme Select the STEP application protocol (AP) to be used for the export.

AP 203 is the protocol for configuration controlled 3D designs of mechanical parts and assemblies.

AP 214 is the protocol for core data for automotive mechanical design processes.

Enable STEP Compound merge If checked, a compound merge will be done during file reading. This is slower but results in higher details.
Use LinkGroup Select this to use App::LinkGroup group containers instead of App::Part group containers.
Import invisible objects Select this to import invisible objects.
Reduce number of objects Reduce the number of objects using Link arrays.
Expand compound shape Expand compound shapes with multiple solids.
Show progress bar when importing Show a progress bar when importing.
Ignore instance names Do not use instance names. Useful for some legacy STEP files with non-meaningful auto-generated instance names.

SVG

Scalable Vector Graphics (SVG) is a vector image format for two-dimensional graphics. A vector image can be scaled to any size without losing its shape or details. An SVG image can be converted to bitmap formats like PNG or JPEG for printing.

On this page you can specify the following:

Name Description
Import style Select how SVG object colors and line widths will be imported.

If None (fastest) is selected no color or line width settings will be imported.

If Use default color and linewidth is selected FreeCAD will use its default color and line width.

If Original color and linewidth is selected FreeCAD will use the color and linewidth from the SVG objects.

Disable units scaling If checked, no unit conversion will occur. One unit in the SVG file will translate as one millimeter.
Export style Select how Sketches are exported to SVG.

If Translated (for print & display) is selected, SVG objects are encapsulated in a group that is scaled and moved to the correct place in the SVG document to fit into a printable area.

If Raw (for CAM) is selected, SVG objects are placed as they are - at the same coordinates as in the FreeCAD model (1:1 export).

Translate white line color to black If checked, all white lines will appear in black in the SVG for better readability against white backgrounds.
Max segment length for discretized arcs Versions of Open CASCADE older than version 6.8 don't support arc projection. In this case arcs will be discretized into small line segments. This value is the maximum segment length.

VTK

The Visualization Toolkit (VTK) is an open-source, freely available software system for 3D computer graphics, image processing and visualization. VTK files are used by the FEM Workbench for the post processing of simulation results.

This page is only shown if the FEM Workbench has been loaded in the current FreeCAD session.

On this page you can specify the following:

Name Description
Which object to import into Select which objects should be imported and how.

If VTK result object is selected, a FreeCAD FEM VTK result object will be imported (equals to the object which was exported).

If FEM mesh object is selected, the results in the VTK file will be omitted, only the mesh data will be imported and a FreeCAD FEM mesh object will be created.

If FreeCAD result object is selected, the imported data will be converted into a FreeCAD FEM Result object. Note: this setting needs the exact result component names and thus it only works properly with VTK files exported from FreeCAD.

En los dos últimos capítulos, vimos cómo crear geometrías de piezas y crear objetos paramétricos. Falta un último elemento para obtener el control total de FreeCAD: crear herramientas que interactúen con el usuario.

En muchas situaciones, no resulta muy práctico crear un objeto con valores cero, como hicimos con el rectángulo en el capítulo anterior, y luego pedirle al usuario que introduzca los valores de altura y anchura en el panel de propiedades. Esto funciona para un número reducido de objetos, pero se vuelve muy tedioso si hay que crear muchos rectángulos. Una mejor solución sería poder especificar la altura y la anchura al crear el rectángulo.

Python ofrece una herramienta básica para que el usuario introduzca texto en la pantalla: 

texto = raw_input("¿Altura del rectángulo?")

print ("La altura ingresada es ",texto)

Sin embargo, esto requiere que la consola de Python esté en funcionamiento, y cuando ejecutamos nuestro código desde una macro, no siempre podemos estar seguros de que la consola de Python esté activada en la máquina del usuario.

La [1] (Interfaz Gráfica de Usuario), que abarca los menús, las barras de herramientas, la vista 3D y otros componentes visuales de FreeCAD, está diseñada para que el software sea intuitivo y accesible. Sirve de puente entre el usuario y la funcionalidad subyacente de FreeCAD, permitiendo que tanto usuarios ocasionales como expertos interactúen con el programa de manera efectiva.

La interfaz gráfica de usuario (GUI) de FreeCAD está construida con Qt, un potente kit de herramientas de código abierto que ofrece una amplia gama de funciones. Qt proporciona elementos básicos para el diseño de interfaces, como cuadros de diálogo, botones, etiquetas, campos de entrada de texto y menús desplegables, conocidos colectivamente como "widgets". Estos widgets constituyen la base de la experiencia de usuario de FreeCAD.

Una de las principales ventajas de Qt es su compatibilidad multiplataforma, que permite que FreeCAD funcione sin problemas en diferentes sistemas operativos como Windows, macOS y Linux. Además, la flexibilidad de Qt facilita a los desarrolladores la ampliación o personalización de la interfaz de FreeCAD, ya sea creando nuevas barras de herramientas y menús o desarrollando módulos completamente nuevos que se integran en el software. Esta adaptabilidad garantiza que FreeCAD siga siendo fácil de usar y altamente extensible.

Para los usuarios interesados ​​en la programación o el desarrollo de nuevas herramientas, la API de Python de FreeCAD también ofrece acceso a numerosas funciones de Qt. Esto significa que no solo se pueden automatizar tareas, sino también crear widgets o diálogos personalizados que se integran directamente en el entorno de FreeCAD.

Las herramientas de Qt son muy fáciles de usar desde Python, gracias a un módulo de Python llamado PySide. PySide es la interfaz oficial de Python para la biblioteca Qt, que proporciona una forma sencilla de crear e interactuar con widgets mediante programación. Permite a los desarrolladores diseñar interfaces, gestionar la entrada de datos del usuario (como leer texto de campos de entrada) y definir acciones basadas en las interacciones del usuario, como responder a la pulsación de un botón. Con PySide, puede crear cuadros de diálogo, menús y barras de herramientas personalizados directamente en FreeCAD, ampliando su funcionalidad de forma que se integre perfectamente con su interfaz existente.

PySide facilita la conexión de las acciones del usuario con funciones específicas de su código. Por ejemplo, puede configurar un botón para que, al pulsarlo, ejecute un script que realice un comando o modifique un objeto en la vista 3D. Esta capacidad interactiva abre un sinfín de posibilidades para personalizar flujos de trabajo y automatizar tareas repetitivas.

Qt también proporciona otra herramienta interesante llamada Qt Designer, que actualmente está integrada en una aplicación más grande llamada Qt Creator. Esta última permite diseñar cuadros de diálogo y paneles de interfaz gráficamente, en lugar de tener que programarlos manualmente. En este capítulo, usaremos Qt Creator para diseñar un widget de panel que utilizaremos en el panel Tareas de FreeCAD. Por lo tanto, deberá descargar e instalar Qt Creator desde su página oficial si usa Windows o Mac (en Linux, generalmente estará disponible desde su gestor de software).

En el siguiente ejercicio, primero crearemos un panel con Qt Creator que solicita valores de longitud, anchura y altura. A continuación, crearemos una clase Python que leerá los valores introducidos por el usuario en el panel y creará un recuadro con las dimensiones especificadas. Esta clase Python será utilizada posteriormente por FreeCAD para mostrar y controlar el panel de tareas.

Comencemos creando el widget. Inicie Qt Creator y luego vaya al menú Archivo → Nuevo archivo o proyecto → Qt → Formulario de Qt Designer → Diálogo sin botones. Haga clic en Siguiente, asígnele un nombre de archivo para guardarlo, haga clic en Siguiente, deje todos los campos del proyecto con su valor predeterminado ("<ninguno>") y haga clic en Crear. El sistema de tareas de FreeCAD agregará automáticamente los botones Aceptar/Cancelar, por eso elegimos un diálogo sin botones.

  • Find the Label in the list in the left panel (under the Display Widgets section), and drag it onto the canvas of our widget. Double-click the recently placed Label, and change its text to Length.
  • Right-click the widget canvas, and choose Lay out → Lay out in a Grid. This will turn our widget into a grid with currently only one cell, occupied by our first label. We can now add the next items at the left, right, top or bottom of our first label, and the grid will expand automatically.
  • Add two more labels below the first one, and change their text to Width and Height:

  • Now place 3 Double Spin Box widgets (under the Input Widgets section) next to our Length, Width and Height labels. For each of them, in the lower right panel, which shows all the available settings for the selected widget, locate Suffix and set their suffix to mm. FreeCAD has a more advanced widget, that can handle different units, but that is not available in Qt Creator by default (but can be compiled), so for now we will use a standard Double Spin Box, and we add the "mm" suffix to make sure the user knows in which units they work:

  • Now our widget is done, we just need to make sure of one last thing. Since FreeCAD will need to access that widget and read the Length, Width and Height values, we need to give proper names to those widgets, so we can easily retrieve them from within FreeCAD. Click each of the Double Spin Boxes, and in the upper right window, double-click their Object Name, and change them to something easy to remember, for example: BoxLength, BoxWidth and BoxHeight:

  • Save the file, you can now close Qt Creator, the rest will be done in Python.
  • Open FreeCAD and create a new macro from menu Macro → Macros → Create
  • Paste the following code. Make sure you change the file path to match where you saved the .ui file created in QtCreator:
import FreeCAD,FreeCADGui,Part
 
# CHANGE THE LINE BELOW
path_to_ui = "C:\Users\yorik\Documents\dialog.ui"
 
class BoxTaskPanel:
   def __init__(self):
       # this will create a Qt widget from our ui file
       self.form = FreeCADGui.PySideUic.loadUi(path_to_ui)
 
   def accept(self):
       length = self.form.BoxLength.value()
       width = self.form.BoxWidth.value()
       height = self.form.BoxHeight.value()
       if (length == 0) or (width == 0) or (height == 0):
           print("Error! None of the values can be 0!")
           # we bail out without doing anything
           return
       box = Part.makeBox(length,width,height)
       Part.show(box)
       FreeCADGui.Control.closeDialog()
        
panel = BoxTaskPanel()
FreeCADGui.Control.showDialog(panel)

In the code above, we used a convenience function PySideUic.loadUi from the FreeCADGui module. That function loads a .ui file, creates a Qt Widget from it, and maps names, so we can easily access the subwidget by their names (ex: self.form.BoxLength).

The "accept" function is also a convenience offered by Qt. When there is an "OK" button in a dialog (which is the case by default when using the FreeCAD Tasks panel), any function named "accept" will automatically be executed when the "OK" button is pressed. Similarly, you can also add a "reject" function which gets executed when the "Cancel" button is pressed. In our case, we omitted that function, so pressing "Cancel" will do the default behaviour (do nothing and close the dialog).

If we implement any of the accept or reject functions, their default behaviour (do nothing and close) will not occur anymore. So we need to close the Task panel ourselves. This is done with: 

FreeCADGui.Control.closeDialog()

Once we have our BoxTaskPanel that has 1- a widget called "self.form" and 2- if needed, accept and reject functions, we can open the task panel with it, which is done with these two last lines: 

panel = BoxTaskPanel()
 FreeCADGui.Control.showDialog(panel)

Note that the widget created by PySideUic.loadUi is not specific to FreeCAD, it is a standard Qt widget that can be used with other Qt tools. For example, we could have shown a separate dialog box with it. Try this in the Python Console of FreeCAD (using the correct path to your .ui file of course): 

from PySide import QtGui
 w = FreeCADGui.PySideUic.loadUi("C:\Users\yorik\Documents\dialog.ui")
 w.show()

Of course, we didn't add any "OK" or "Cancel" buttons to our dialog, since it was made to be used from the FreeCAD Task panel, which already provides such buttons. So there is no way to close the dialog (other than pressing its Window Close button). But the function show() creates a non-modal dialog, which means it doesn't block the rest of the interface. So, while our dialog is still open, we can read the values of the fields: 

w.BoxHeight.value()

This is very useful for testing.

Finally, don't forget there is much more documentation about using Qt widgets on the FreeCAD Wiki, in the Python Scripting section, which contains a dialog creation tutorial, a special 3-part PySide tutorial that covers the subject extensively.

¿Qué es un Imágenes de las aplicaciones?

Empaquetar una vez y correr por todas partes. Llega a los usuarios en todas las principales distribuciones de escritorio de Linux.

AppImage es un "paquete binario universal" diseñado para distribuir una aplicación a cualquier distribución de Linux. Para obtener más información, visite la página principal de AppImage en [1] y la página de Wikipedia en [2].

Para ejecutarlo, primero hágalo ejecutable y luego escriba la ruta relativa o completa. 

chmod +x FreeCAD_xxx-x86_64.AppImage
./FreeCAD_xxx-x86_64.AppImage

Para otros tipos de instalación, consulte Descargar.

Notas importantes:

  • El desarrollo es diario y rápido.
  • Muchos usuarios del foro utilizan la versión de desarrollo.
  • Puede ejecutarse en el mismo sistema en paralelo con otra versión de FreeCAD.
  • Los usuarios utilizan la versión de desarrollo para aprovechar las últimas funciones y correcciones de errores (dado que FreeCAD tiene un ciclo de lanzamiento prolongado). También la utilizan para probar y detectar errores, lo que impulsa el desarrollo y la mejora de FreeCAD.

Advertencia obligatoria

En general, la versión de desarrollo es estable, pero, por supuesto, es importante añadir la advertencia obligatoria de que su uso es bajo su propia responsabilidad. Sin embargo, la mayoría de las personas que realizan copias de seguridad y guardan sus diseños con frecuencia no tienen mayores problemas.

Actualización automática

AppImage utiliza un método inteligente y económico para actualizarse. Calcula la diferencia entre la nueva AppImage y la anterior, y solo descarga los cambios entre ambas versiones. En teoría, el usuario termina descargando alrededor del 15 % cada vez, en lugar de una AppImage completamente nueva.

La actualización automática se realiza mediante varios métodos opcionales. Actualmente existen 4 métodos: 2 a través de la interfaz gráfica de usuario (GUI) y 2 a través de la interfaz de línea de comandos/terminal (CLI).

Actualización Experimental dentro de la Aplicación

Gracias a los esfuerzos de varios desarrolladores clave, se está llevando a cabo un esfuerzo continuo para integrar una función que permita actualizar automáticamente la AppImage dentro del propio FreeCAD. A partir de FC 0.19.21514, ahora existe una sección AppImage a la que se accede mediante Editar → Preferencias → AppImage. Pruebe esta función e informe de su experiencia en el debate del foro.

Método GUI 1 (oficial)

Esta es la aplicación oficial de AppImageUpdate con interfaz gráfica.

  1. Descargue AppImageUpdate-x86_64.AppImage.
  2. Hágala ejecutable haciendo clic derecho en el archivo, seleccionando Propiedades y luego Ejecutar como ejecutable.
  3. Haga doble clic en el icono de AppImage. Aparecerá un cuadro de diálogo donde se le pedirá que especifique qué AppImage desea actualizar.
  4. Especifique la ruta a su AppImage existente.
  5. Una vez actualizada la AppImage, pulse el botón Ejecutar AppImage Actualizada.

Método GUI 2 (no oficial)

Esta es una versión no oficial de terceros, más elegante, de AppImageUpdate, llamada AppImageUpdater. Aún está en desarrollo (al momento de editar esta wiki), pero aun así, es bastante fácil de usar.

  1. Descargar AppImageUpdater-*-x86_64.AppImage
  2. Hacerlo ejecutable: 
    chmod +x AppImageUpdater*-x86_64.AppImage
  3. Ejecutarlo: 
    source AppImageUpdater*-x86_64.AppImage
  4. Localizar su archivo AppImage de FreeCAD y arrastrarlo sobre AppImageUpdater.

Resultado: Siga las instrucciones de AppImageUpdater.

Método CLI 1 (oficial)

Ejecute las siguientes instrucciones desde la terminal: 

wget https://github.com/AppImage/AppImageUpdate/releases/download/continuous/appimageupdatetool-x86_64.AppImage
chmod +x ./appimageupdatetool-x86_64.AppImage
./appimageupdatetool.AppImage path/to/old/FreeCAD.AppImage
chmod +x path/to/updated/FreeCAD.AppImage
./path/to/updated/FreeCAD.AppImage

Notas: * Los nombres de archivo serán únicos debido a que la información de la versión está integrada en ellos. Las instrucciones anteriores se han simplificado para mayor comodidad.

  • Ejecute ./appimageupdatetool-x86_64.AppImage --help para obtener información sobre funciones como --remove-old, --overwrite y --self-update.
  • También existe una versión para i386; consulte la página de lanzamientos de AppImageUpdate en [3].

Pendiente: Compartir un script que se pueda agregar como alias o cron tarea.

Método CLI 2 (no oficial)

De manera similar a los métodos gráficos, que cuentan con enfoques oficiales y no oficiales para descargar AppImages, ocurre con la línea de comandos. Esta es una opción de línea de comandos de terceros más elegante para descargar AppImages.

  1. Descargue appimageupdater-*-x86_64.AppImage
  2. Hágalo ejecutable: 
    chmod +x appimageupdater*-x86_64.AppImage
  3. Ejecútelo: 
    source appimageupdater*-x86_64.AppImage /ruta/al/archivo/antiguo/FreeCAD-AppImage.AppImage

Resultado: Actualice el archivo AppImage especificado si existe una actualización.

Solucionando el problema de zsync en AppImage

Puede ocurrir que una AppImage no se actualice porque su archivo de destino haya cambiado. En lugar de descargar una AppImage completamente nueva, es posible sobrescribir el archivo zsync que utiliza la AppImage para descargar la actualización delta. Para más información, consulte https://github.com/antony-jr/appimage-update-info-writer.

Sección para Desarrolladores

Nota: Las siguientes secciones están destinadas a desarrolladores

Desempaquetando de AppImages

Una gran ventaja de FreeCAD es que la mayor parte está escrita en Python, lenguaje que no requiere compilación manual como C++. Básicamente, se puede modificar un archivo Python y, al reiniciar FreeCAD, esos cambios se integrarán en la aplicación. Un desarrollador puede trabajar rápidamente con la última versión de FreeCAD utilizando esta técnica y una AppImage. Además, usar una AppImage no modifica el entorno del sistema; es decir, no se instala nada ni se modifican las variables de entorno.

Modificación de AppImages

Un archivo AppImage contiene un sistema de archivos con todo lo necesario para ejecutar la aplicación. Para modificarlo, es necesario extraer dicho sistema de archivos. 

./FreeCAD_xxx.AppImage --appimage-extract
cd squashfs-root/

Ahora abra los archivos fuente de Python necesarios en su editor de código preferido, modifíquelos y guárdelos. Luego ejecute la aplicación. 

./AppRun

Reempaquetado de AppImages

Si ha modificado el código y ahora desea volver a empaquetar la AppImage con sus últimos cambios, utilice la herramienta appimagetool-x86_64 en el sistema de archivos extraído. 

cd ..
wget "https://github.com/AppImage/AppImageKit/releases/download/continuous/appimagetool-x86_64.AppImage"
chmod +x appimagetool-x86_64.AppImage
./appimagetool-x86_64.AppImage squashfs-root

AppImages Personalizadas

Gracias al trabajo de realthunder, autor de Enlace de la Aplicación y Entorno de Trabajo de Ensamblado3, es posible crear AppImages personalizadas utilizando un conjunto de scripts.

Esto facilita enormemente la publicación de imágenes para una rama específica del código fuente, permitiendo así que otros las prueben. Si bien las AppImages solo funcionan en Linux, los scripts de realthunder permiten generarlas también en Windows y macOS.

El repositorio de estos scripts se encuentra en [4]. Para obtener más detalles, consulte el archivo [5].
